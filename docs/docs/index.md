# Introduction: AI Project Manager & Portfolio Generator

**Contributors: Heidi Pappila, Joni Kauppinen**

This documentation covers the development of the AI Project Manager & Portfolio Generator. This project was conceived to streamline software project tracking and enhance the visibility of developer skills through intelligent automation.

**Mission and Objectives**

The primary goal of this project is to build an AI-assisted application capable of autonomously analyzing GitHub repository data—including commit history, issues, and code structure. The tool addresses two core challenges:

1. Project Management Overhead: By providing automated status snapshots and suggesting subsequent tasks, the system helps developers and small teams maintain momentum.

2. Documentation Gaps: The system generates professional README files and portfolio content, ensuring a developer's technical expertise is clearly communicated without the burden of manual writing.

**Client and Scope**

The project is carried out as a student assignment, specifically targeting students and small development teams. The project scope is defined as 5 ECTS credits, representing a total workload of approximately 200 hours. The development timeline is scheduled from January 31, 2026, to March 31, 2026.

**Technical Framework**

The solution is built using a modern technical stack:

 * Backend: Python & FastAPI
 * Frontend: Streamlit
 * AI Models: Gemini Flash or Llama (integrated via GitHub API)
 * Infrastructure: SQLite database and Docker containerization
